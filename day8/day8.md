## なぜRNNが必要か

通常のニューラルネットは固定長の入力しか受け付けない。テキストは文章ごとに長さが違うので対応できない。

**RNNのアイデア：「単語を1つずつ順番に処理して前の情報を次に引き継ぐ」**

```
x1(犬) → [RNN] → h1
                  ↓
x2(が) → [RNN] → h2
                  ↓
x3(好き) → [RNN] → h3 → 出力
```

`h`（隠れ状態）が「今まで読んだ内容の記憶」の役割を果たす。同じRNNの重みを各ステップで使い回す。

---

## output・hidden・cellの違い

```python
# RNN
output, hidden = rnn(x)

# LSTM
output, (hidden, cell) = lstm(x)
```

```
output：各ステップのhiddenを全部並べたもの → 系列全体が必要なとき（翻訳など）
hidden：最後のステップのhidden → 文章全体を1つのベクトルで表したいとき（分類など）
cell  ：長期記憶。基本的に内部で引き継ぐだけで外から直接使わない
```

```python
# outputの最後とhiddenは同じ
output[:, -1, :] == hidden  # True

# cellはhiddenとは別物
```

---

## 勾配消失問題

逆伝播（BPTT）は全ステップが終わってから1回だけ実行される。

```
loss → h50 → h49 → ... → h2 → h1
```

各ステップで同じ重みが掛け算され続ける。

```
重みが小さい場合（0.5）：0.5^50 ≈ 0  → 勾配消失
重みが大きい場合（2.0）：2.0^50 → 巨大 → 勾配爆発
```

→ 最初の単語の情報が長い文章では学習されなくなる。

---

## LSTMによる解決

ゲート機構でセル状態（長期記憶）を管理する。

```
忘却ゲート：古い記憶のどれを忘れるか（0〜1で制御）
入力ゲート：新しい情報のどれを記憶するか（0〜1で制御）
出力ゲート：記憶のどれを出力するか（0〜1で制御）

cell_new = 忘却ゲート × cell_old + 入力ゲート × 新情報
```

掛け算ではなく**加算**で情報を引き継ぐので勾配が消えにくい。

### hiddenとcellの違い

```
hidden：毎ステップ大きく変わる（直前の単語に強く引っ張られる・短期記憶）
cell  ：ゆっくり変わる（重要な情報を長く保持する・長期記憶）
```

cellは8次元などの固定サイズで、増えていくのではなく値が更新され続ける。

### cellを直接使う場面

基本的には使わない。長い文書を分割して処理するときに文脈を引き継ぐために使うことがある。

```python
output1, (hidden1, cell1) = lstm(x_part1)
output2, (hidden2, cell2) = lstm(x_part2, (hidden1, cell1))
```

---

## LSTMの限界

```
短い文章（〜20単語）：LSTMで十分
長い文章（100単語〜）：LSTMでも限界がある
```

根本的な解決策はDay 9のAttention機構。「離れた単語間を直接つなぐ」ことで距離の問題を回避する。

## キーワード集

### `nn.RNN` / `nn.LSTM`

```python
rnn = nn.RNN(
    input_size=4,    # 各単語のベクトルの次元数
    hidden_size=8,   # 隠れ状態の次元数
    batch_first=True # 入力の形を(バッチ, 系列長, 特徴量)にする
)

lstm = nn.LSTM(input_size=4, hidden_size=8, batch_first=True)
```

### BPTT（Backpropagation Through Time）
RNN・LSTMの逆伝播。全ステップの順伝播が終わってから1回だけ実行される。時間方向に逆向きにたどって勾配を計算する。

## 疑問・メモ欄

### 重みが一律なのになぜhiddenは直前の単語に強く引っ張られるのか

重みは同じでも「直接足されるか」「何回も掛け算を経由するか」で影響の強さが変わる。

```
h_new = tanh(W_h × h_old + W_x × x_now)

遠い単語：W_h^50 × h_old ≈ 0  （何度も掛け算で消える）
直前の単語：W_x × x_now        （直接足されるので強い）
```

さらに `tanh` は出力が `-1〜1` に収まるので、何ステップも経ると古い情報が押しつぶされていく。
