## キーワード集

### `reshape(-1, 1)`
NumPyの配列の「形」を変える処理。

```python
a = np.array([20, 25, 30, 35, 40])
a.shape        # (5,)   ← 1次元（ただの数字の列）
a.reshape(-1, 1).shape  # (5, 1) ← 5行1列の2次元配列
```

scikit-learnは「表形式のデータ（行×列）」を要求するので変換が必要。  
`-1` は「行数はデータ数に合わせて自動計算して」という意味。

---

### `LinearRegression()`
線形回帰の計算をするオブジェクト（箱）を作る処理。

```python
from sklearn.linear_model import LinearRegression
model = LinearRegression()  # 箱を用意する
```

箱の中には：
- **データ置き場**：`coef_`（傾き）、`intercept_`（切片）← fit後に値が入る
- **機能**：`fit()`、`predict()` などのメソッド

`from sklearn.linear_model import LinearRegression` は  
「sklearn → linear_model → LinearRegression という場所にある設計図を使う」という宣言。

---

### `fit(X, y)`
データを使って傾きと切片を計算し、モデル内部に保存する処理。

```python
model.fit(気温.reshape(-1, 1), 売上)
# 実行後に中身が入る
print(model.coef_)       # 傾き
print(model.intercept_)  # 切片
```

内部では**最小二乗法**（ズレの二乗和を最小化）で計算している。

---

### `predict(X)`
`fit` で求めた傾き・切片を使って予測値を返す処理。

```python
予測値 = model.predict(気温.reshape(-1, 1))

# predictが内部でやっていること（単回帰の場合）
手動予測 = model.coef_[0] * 気温 + model.intercept_
# → 両者は一致する
```

---

### 最小二乗法
「予測値と実際の値のズレの二乗和を最小化する」ことで直線を求める方法。

二乗する理由：ズレにはプラスとマイナスがあり、そのまま足すと打ち消し合うため。

```python
ズレ = 売上 - 予測値
二乗和 = np.sum(ズレ ** 2)  # これを最小化している
```

---

### 単回帰 vs 重回帰

```
単回帰：売上 = 傾き × 気温 + 切片
重回帰：売上 = 傾き1 × 気温 + 傾き2 × 湿度 + 切片
```

説明変数が複数のとき、`np.column_stack` で横に並べて行列にする。

```python
X = np.column_stack([気温, 湿度])
# shape: (5, 2) ← 5行2列
```

`coef_` も説明変数の数だけ出てくる（並び順は `column_stack` に渡した順）。

---

### 標準化
変数ごとにスケール（単位の大きさ）が違うと、傾きの大きさをそのまま比べられない。

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

標準化すると各変数が「平均0、標準偏差1」に揃う。  
これにより傾きの大きさ＝影響の強さとして比較できるようになる。

---

### 多重共線性
説明変数同士が強く相関していると起きる問題。

**本質的な問題は「解釈」**：

| 問題 | 内容 |
|------|------|
| 予測精度 | あまり下がらない |
| 傾きの解釈 | できなくなる（どちらの変数の影響か分離できない） |
| 傾きの安定性 | データが少ないと暴れやすい |

例：気温と湿度が完全に連動しているとき  
→「気温が1上がると売上が○増える」とは言えない（湿度も同時に動くので）

確認方法：VIF（分散拡大係数）※NLPではあまり登場しない

## 今日の流れまとめ

```
1. 完璧なデータで単回帰 → y = 10x - 100 を手で予測 → 機械も同じ答えを出した
2. ノイズ入りデータ → 傾き・切片が少しズレる（現実はこっち）
3. 説明変数を2つに増やす → 重回帰
4. 傾きの大きさだけで比べると危ない → 標準化で解決
5. 説明変数が連動していると傾きを解釈できない → 多重共線性
```

### 重回帰のとき標準化はすべき？
**目的による。**
 
| 目的 | 標準化 |
|------|--------|
| 変数の影響の大きさを比べたい | する |
| 予測精度だけが目的（線形回帰） | しなくてもいい |
| ニューラルネット・SVM・k-NNを使う | 必ずする |
 
ニューラルネットはスケールの大きい変数に引っ張られる性質があるため。  
NLPではニューラルネットをよく使うので「とりあえず標準化する」習慣をつけておくと良い。
