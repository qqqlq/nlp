## ニューラルネットの本質

**「たくさんの線形回帰を組み合わせたもの」**

```
入力層 → 隠れ層1 → 隠れ層2 → 出力層
```

---

## なぜ活性化関数が必要か

線形変換だけを重ねても結局1つの線形変換になってしまう。

```python
y = W2 @ (W1 @ x)
  = (W2 @ W1) @ x  # W3 = W2@W1 とおけば
  = W3 @ x         # 1層と同じ → 表現力が増えない
```

活性化関数を挟むことで非線形性が生まれ、複雑なパターンを学習できるようになる。

```
線形変換のみ：直線しか引けない
活性化関数あり：複雑な境界線を引ける
```

**「線形変換（どの方向に注目するか）+ 活性化関数（非線形性）」がセットで1つのニューロン**

---

## 活性化関数

### ReLU
```python
def relu(x):
    return np.maximum(0, x)

# x > 0：そのまま通す
# x ≤ 0：0にする
```

厳密には**区分線形関数**（各区間では線形だが全体としては非線形）。  
x=0で微分不可能（劣微分で対処）。  
飽和しないので深いネットワークでも勾配が消えにくい。

### シグモイド
```python
def sigmoid(x):
    return 1 / (1 + np.exp(-x))
```

出力が0〜1に収まる → 確率の表現に向いている。  
両端で傾きがほぼ0になる（**飽和**）→ 深いネットワークでは勾配消失問題が起きる。

---

## ネットワークの構造と行列の形

「入力2つ・隠れ層3ユニット・出力1つ」の場合：

```
入力層          隠れ層         出力層
  x1 ───┐
         ├──→ h1 ──┐
  x2 ───┤    h2    ├──→ y
         ├──→ h3 ──┘
        W1          W2
```

```python
W1 = torch.randn(2, 3)  # 入力2 → 隠れ層3（接続が2×3=6本）
b1 = torch.zeros(3)     # 隠れ層3ユニット分のバイアス
W2 = torch.randn(3, 1)  # 隠れ層3 → 出力1
b2 = torch.zeros(1)     # 出力1ユニット分のバイアス
```

行列積で次元が変換される：
```
入力 (1×2) @ W1 (2×3) → 隠れ層 (1×3)
隠れ層 (1×3) @ W2 (3×1) → 出力 (1×1)
```

---

## 順伝播

```python
z1 = x @ W1 + b1    # 線形変換
h1 = torch.relu(z1) # 活性化関数
z2 = h1 @ W2 + b2   # 線形変換
y_pred = torch.sigmoid(z2)  # 活性化関数（確率に変換）
```

ReLUは配列のどんな形でも**要素ごとに適用**される（ブロードキャスト）。

## 損失関数

予測値と正解のズレを数値化する。

```python
# 二乗誤差
loss = torch.mean((y_true - y_pred) ** 2)
```

## 逆伝播と自動微分

PyTorchは `requires_grad=True` を指定すると計算過程を記録する。

```python
W1 = torch.randn(2, 3, requires_grad=True)

loss.backward()  # この1行で全ての勾配を自動計算
print(W1.grad)   # W1の勾配
```

---

## 勾配降下法

```
新しい重み = 現在の重み - 学習率 × 勾配
```

学習率：一歩の大きさ。大きすぎると発散、小さすぎると収束が遅い。一般的に0.01や0.001から始める。

```python
lr = 0.01
with torch.no_grad():  # 更新時は勾配計算を止める
    W1 -= lr * W1.grad
```

---

## 学習ループの正しい流れ

```python
for i in range(100):
    # 1. 順伝播
    y_pred = ...
    loss = ...

    # 2. 勾配をリセット（前回の累積を消す）
    W1.grad = None

    # 3. 逆伝播
    loss.backward()

    # 4. 重みを更新
    with torch.no_grad():
        W1 -= lr * W1.grad
```

**勾配をリセットする理由**：PyTorchは `backward()` のたびに勾配を累積する仕様のため。リセットしないと勾配がどんどん足し算されてしまう。

## キーワード集

### テンソル（Tensor）
PyTorchが扱う多次元配列。NumPy配列とほぼ同じだが「計算過程を記録できる」点が決定的な違い。

```
0次元：スカラー  tensor(3.14)
1次元：ベクトル  tensor([1., 2., 3.])
2次元：行列      tensor([[1., 2.], [3., 4.]])
3次元以上：テンソル
```

### `requires_grad=True`
「この変数の勾配を計算してね」とPyTorchに指示するフラグ。

### `loss.backward()`
逆伝播を実行して全変数の勾配を自動計算する。連鎖律（chain rule）を内部で使っている。

### `torch.no_grad()`
勾配計算を一時的に止める。重みの更新時に使う。

### `@` 演算子
行列積。`np.dot()` と同じ意味。

### 死んだニューロン
ReLUで0になったユニットは出力に影響しないので勾配も0になる。そのユニットへの接続は学習されない。

---

## 今日の流れまとめ

```
1. 線形変換だけでは表現力が増えない → 活性化関数が必要
2. ReLU：非線形・飽和しない。シグモイド：確率に向く・飽和する
3. 行列積で次元を変換しながら順伝播
4. loss.backward() で勾配を自動計算
5. 学習率 × 勾配で重みを更新
6. 100回繰り返すと損失が 0.0215 → 0.0068 まで下がった
```

### 勾配リセットと計算グラフの関係

W1に紐づいているものは3つある：

```
① W1の値そのもの      → 更新されるが消えない
② 計算グラフ（過程）  → forward時に毎回新しく作られる
③ W1.grad（勾配）     → backward()のたびに累積される → リセットが必要
```

`W1.grad = None` でリセットするのは③だけ。①②は残ったまま。  
計算グラフ（②）はforwardのたびに自動で作り直されるので、勾配をリセットしても `loss.backward()` で新しい勾配を計算できる。

```python
for i in range(100):
    y_pred = ...        # ②計算グラフが新しく作られる
    loss = ...

    W1.grad = None      # ③勾配だけリセット（①②は残ったまま）
    loss.backward()     # ②を逆向きにたどって③を計算
    W1 -= lr * W1.grad  # ①を更新
```
ReLUは配列のどんな形でも要素ごとに処理してくれる。
`np.maximum(0, x)` も `torch.relu(x)` も内部でブロードキャストが効いている。
