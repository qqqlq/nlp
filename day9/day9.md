## なぜAttentionが必要か

```
RNN：私 → が → 好き → な → 犬（順番に処理・距離が遠いと情報が薄れる）
Attention：「私」から全単語に直接アクセスできる（距離の概念がない）
```

---

## Q・K・Vの直感（YouTube検索の例）

```
Q（Query）：あなたが検索する言葉     ← 「何を探しているか」
K（Key）  ：動画のタイトル・タグ     ← 「自分はどんな情報か」
V（Value）：動画の実際の内容        ← 「実際に渡す情報」

Q × K のマッチング → スコア計算
スコアを重みにして V を混ぜ合わせる → 検索結果
```

Attentionに当てはめると：

```
「犬」をQueryとしたとき
Q（犬が探しているもの）× K（各単語のラベル）→ スコア
  「私」：スコア高い（主語として関係ある）
  「が」：スコア低い（助詞なので関係薄い）

output = 「私」のV × 高スコア + 「が」のV × 低スコア + ...
→ 「文脈を考慮した犬の表現」
```

---

## なぜQ・K・Vを別々に作るのか

同じXを3つの視点に変換することで柔軟性が生まれる。

```
X をそのまま使う → 「探す視点」「ラベルの視点」「情報の視点」が全部同じ

W_Q・W_K・W_V で別々に変換 →
  「探すときはこの特徴を見る」
  「ラベルとしてはこの特徴を使う」
  「情報としてはこの特徴を渡す」
  → 3つの役割を別々に学習できる
```

W_Q・W_K・W_V は学習で更新される重み。

---

## 全体のフロー

```
① 入力X（5×4）← 各単語のベクトル（本来はWord2Vecなどの埋め込み）

② Q = X @ W_Q  ← 「何を探しているか」の視点に変換
   K = X @ W_K  ← 「自分はどんな情報か」の視点に変換
   V = X @ W_V  ← 「実際に渡す情報」に変換

③ scores = Q @ K.T / √d_k  ← 各単語ペアのスコア計算（5×5）

④ weights = softmax(scores)  ← スコアを確率に変換（各行の合計=1）

⑤ output = weights @ V  ← 文脈を考慮した表現を生成（5×4）
```

---

## Attentionスコアとコサイン類似度の関係

```python
# コサイン類似度
cos_sim = np.dot(v1, v2) / (norm(v1) * norm(v2))

# Attentionスコア
score = Q @ K.T / √d_k
```

本質的には同じで「QとKの向きが近いほどスコアが高くなる」点で一致している。  
コサイン類似度はベクトルの長さで正規化、Attentionスコアは√d_kで正規化。

## キーワード集

### `K.T`
転置。行と列を入れ替える。

```python
K.shape    # (5, 4)
K.T.shape  # (4, 5)
```

行列積の形を合わせるために使う。

### `/ √d_k`
スケーリング。内積の値が次元数が大きいほど大きくなりすぎるので√d_kで割って安定させる。

```python
d_k ** 0.5  # √d_k
```

### `F.softmax(scores, dim=-1)`
各行の合計が1になるように変換する。`dim=-1` は「最後の次元方向に適用する」という意味。

### `detach()`
計算グラフから切り離してただの数値として扱う。表示・確認のためだけに値を見たいときに使う。逆伝播には影響しない。

```python
output.detach().round(decimals=3)  # 表示するだけ
```

## 今回のQ・K・Vは適当、本来は？

```
今回：W_Q・W_K・W_Vをランダムで初期化 → 意味のないQ・K・V

本来：W_Q・W_K・W_Vも学習で更新される
     大量のテキストから「犬をQueryにしたとき私のKeyのスコアが高くなる」
     という重みを学習していく
```

BERTなどの学習済みモデルではW_Q・W_K・W_Vが大量のテキストから学習済み。

---

## RNN vs Attention まとめ

```
RNN：距離が遠い単語の情報が薄れる
     → 50単語先の情報はほぼ失われる

Attention：全単語に直接アクセス
           → 距離の概念がない
           → 1単語目と50単語目を直接つなげる
```
